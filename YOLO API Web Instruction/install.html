DARKNET 설치 방법

Darknet은 다음과 같이 두가지 방식을 사용하면 설치하기 쉽습니다.

  • OpenCV : 다양하게 지원되는 이미지 유형을 원할 경우
  • CUDA : CPU 계산을 원할 경우

둘 다 선택 사항이므로 기본 시스템을 설치하는 것만으로 시작할 수 있으며 테스트 환경은 리눅스와 맥 환경에서만 실시하였습니다.

기본 시스템 설치

첫째로. Darknet git 저장소를 여기서 다운해 주세요. 다음 명령어들을 참고하시면 됩니다.

git clone https://github.com/pjreddie/darknet.git
            cd darknet
            make

이러한 작업을 수행하면 다음과 같은 많은 수집 정보를 모아야 합니다. 

mkdir -p obj
            gcc -I/usr/local/cuda/include/  -Wall -Wfatal-errors  -Ofast....
            gcc -I/usr/local/cuda/include/  -Wall -Wfatal-errors  -Ofast....
            gcc -I/usr/local/cuda/include/  -Wall -Wfatal-errors  -Ofast....
            .....
            gcc -I/usr/local/cuda/include/  -Wall -Wfatal-errors  -Ofast -lm....

모든 수집 정보들을 모은것이 확인됬다면 프로그램을 실행해 보세요! 

./darknet

다음과 같은 출력 결과를 얻어야합니다.

usage: ./darknet <function>

좋습니다! 이제 여기서 여러분이 할 수 있는 일들을 확인해 보세요.

CUDA와 협업

CPU에 있는 Darknet은 빠르지만 GPU에서 500배 더 빠릅니다! 그러기에 Nvidia GPU 와 CUDA 설치해야 합니다. CUDA 설치에 관해서는 복잡한 내용이기에 깊게 다루지는 않겠습니다.

CUDA가 설치되어 있다면, 기본 디렉토리에서 Makefile의 첫번째 행을 읽기 전용으로 바꿔줍니다.

GPU=1

이제 여러분은 make 입력하면 프로젝트를 생성하고 CUDA를 활성화 할 수 있습니다. 기본적으로 시스템의 우선 순위로 그래픽 카드를 네트워크에서 실행합니다 (만약 CUDA가 올바르게 설치됬다면 nvidia-smi 이용하여 당신의 그래픽 카드 리스트를 확인할 수 있습니다). 만약 당신이 Darknet을 이용하는데 카드를 바꾸기를 원한다면 명령어 줄의 플래그 -i <index>를 아래를 참고하여 바꾸시면 됩니다.

./darknet -i 1 imagenet test cfg/alexnet.cfg alexnet.weights

만약 여러분이 CUDA를 사용하여 컴파일 할 수 있지만 어떤 이유로든 사용할 수 있는 CPU계산을 위해 CPU를 대신하여-nogpu 명령어를 사용하세요.

./darknet -nogpu imagenet test cfg/alexnet.cfg alexnet.weights

Enjoy your new, super fast neural networks!

OpenCV를 이용한 컴파일

기본적으로 Darknet은 이미지 로딩을 위해 stb_image.h 를 사용합니다. 또한 OpenCV를 사용하면 디스크에 저장하지 않아도 이미지와 탐지 기능을 볼 수 있습니다.

우선 OpenCV를 먼저 설치하세요. 소스에서 이 작업을 수행하면 오래걸리고 복잡하므로 패키지 매니저를 사용하여 수행할 수 있습니다.

다음으로 Makefile의 두번째 줄을 변경합니다.

OPENCV=1

다 끝났습니다! 실행하기 전, 우선 다시 make를 사용해 프로젝트를 다시 시작하세요. 그 다음 imtest 사용하여 영상 로드를 테스트하고 다음을 표시합니다.

./darknet imtest data/eagle.jpg
